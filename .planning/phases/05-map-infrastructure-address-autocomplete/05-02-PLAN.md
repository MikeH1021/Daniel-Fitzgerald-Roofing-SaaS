  <action>
    **1. Add `fetchMapsKey` to `packages/widget/src/api/client.ts`:**
    - Add function: `export async function fetchMapsKey(): Promise<string>`
    - Fetches `${apiBase}/api/maps/key`, throws if not ok, returns `data.key`

    **2. Create `packages/widget/src/components/AddressAutocomplete.tsx`:**
    - Props: `{ onPlaceSelected: (place: SelectedPlace) => void }`
    - Internal state: `inputValue` signal (string), `showDropdown` signal (boolean)
    - `useRef` for input element (to position portal dropdown)
    - On input change: update `inputValue`, debounce 300ms using `useRef`-based timer, then call `fetchSuggestions(inputValue)` and set `suggestions` signal from state/map.ts
    - Render: `<input>` with type="text", placeholder="Search for your address...", class="rc-address-input"
    - On input blur: delay 150ms then set `showDropdown = false` (allows click to register)
    - On input focus: if suggestions exist, set `showDropdown = true`
    - **Dropdown:** Use `createPortal` from `preact/compat` to render suggestion list into `document.body`
      - Position with `position: fixed`, calculated from input's `getBoundingClientRect()`
      - Use `onMouseDown` with `e.preventDefault()` on each `<li>` to prevent blur-before-click (RESEARCH.md Pitfall 5)
      - On suggestion click: call `resolvePlaceLocation(suggestion)`, set `selectedPlace` signal, set `showDropdown = false`, call `onPlaceSelected(place)`
    - Style dropdown inline: white bg, border, border-radius 6px, box-shadow, z-index 999999
    - Each suggestion `<li>`: padding 10px 14px, cursor pointer, hover state via onMouseEnter/Leave toggling bg color

    **3. Create `packages/widget/src/components/MapView.tsx`:**
    - Props: `{ lat: number, lng: number }`
    - `useRef` for map container div
    - `useEffect` on lat/lng: call `importMapsLibrary('maps')`, get `Map` constructor, initialize with:
      - `center: { lat, lng }`
      - `zoom: 20` (roof-level)
      - `mapTypeId: 'hybrid'` (satellite with labels)
      - `disableDefaultUI: true`
      - `gestureHandling: 'greedy'` (mobile single-finger pan)
    - Store map instance in `useRef` — if map already exists, call `map.setCenter({ lat, lng })` instead of creating new
    - Render: `<div ref={mapRef} class="rc-map-container" style={{ width: '100%', height: '300px' }} />`
    - The explicit height is critical (RESEARCH.md Pitfall 3 — Maps API renders 0px without it)

    **4. Create `packages/widget/src/components/MapStep.tsx`:**
    - Orchestrates the map flow within the RoofDetails step
    - Props: none (reads from map state signals)
    - On mount (when `mapMode` becomes true):
      - Set `mapLoading = true`
      - Call `fetchMapsKey()` to get API key, set `apiKey` signal
      - Call `loadMapsApi(apiKey)` to inject bootstrap loader
      - Set `mapLoading = false`
    - Render:
      - If `mapLoading`: show "Loading map..." spinner/text
      - If not loaded yet: show nothing
      - Once loaded: show `<AddressAutocomplete onPlaceSelected={handlePlaceSelected} />`
      - If `selectedPlace` is set: show `<MapView lat={...} lng={...} />` below the autocomplete
      - Show the formatted address as text below autocomplete when a place is selected
    - `handlePlaceSelected`: sets `selectedPlace` signal

    **5. Update `packages/widget/src/App.tsx`:**
    - Import `mapMode` from `../state/map`
    - Import `MapStep` component
    - In the `currentStep.value === 0` block (RoofDetails step), add a "Measure on map" link/button below or near the sqft field:
      - Clicking toggles `mapMode.value = true`
      - When `mapMode` is true, render `<MapStep />` above or replacing the sqft manual input
      - Add a "Enter sqft manually" link to toggle back (`mapMode.value = false`)
    - The toggle should NOT change `currentStep` — map mode is a sub-mode within step 0
    - Keep the existing RoofDetails flow fully functional when mapMode is false (MAP-03: no Maps loading until activated)

    **6. Add styles to `packages/widget/src/styles/widget.css`:**
    - `.rc-address-input`: full width, padding 10px 14px, border 1px solid #d1d5db, border-radius 6px, font-size 15px, outline focus state
    - `.rc-map-container`: width 100%, border-radius 6px, margin-top 12px, overflow hidden
    - `.rc-map-toggle`: cursor pointer, color var(--rc-primary), text-decoration underline, font-size 14px, margin-top 8px, display inline-block
    - `.rc-map-loading`: text-align center, padding 20px, color #6b7280
    - `.rc-selected-address`: font-size 13px, color #374151, margin-top 4px, padding 4px 0

    **Important implementation notes:**
    - The portal dropdown renders into `document.body` (outside Shadow DOM) — it needs inline styles, NOT CSS classes (Shadow DOM styles do not reach document.body)
    - The `mapMode` signal controls lazy loading — `fetchMapsKey` and `loadMapsApi` must NOT be called until `mapMode` becomes true (MAP-03 requirement)
    - Do not import any Google Maps npm packages — all Maps interaction is via the CDN loader from Plan 01

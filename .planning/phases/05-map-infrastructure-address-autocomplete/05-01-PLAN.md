    - loadMapsApi(key) returns the same promise on second call (singleton)
    - loadMapsApi(key) resolves immediately if google.maps.importLibrary already exists on window
    - importMapsLibrary(name) calls google.maps.importLibrary(name)
    - fetchSuggestions(input) creates a new AutocompleteSessionToken on first call
    - fetchSuggestions(input) reuses existing session token on subsequent calls
    - fetchSuggestions(input) returns array of PlacePredictionResult with text, placeId, mainText, raw
    - resolvePlaceLocation(suggestion) calls toPlace().fetchFields() and returns SelectedPlace
    - resolvePlaceLocation(suggestion) sets sessionToken to null after completion
  </behavior>
  <action>
    1. Create `packages/widget/src/maps/loader.ts`:
       - `loadMapsApi(apiKey: string): Promise<void>` — singleton promise pattern
       - First check: if `google?.maps?.importLibrary` exists, resolve immediately (host page already loaded Maps)
       - Otherwise: create `<script>` element with `textContent` set to the Google bootstrap loader snippet (from RESEARCH.md Pattern 1), interpolating `apiKey`
       - Append to `document.head`, then `resolve()` immediately (inline scripts execute synchronously on append — do NOT wait for onload)
       - Store promise in module-level `loaderPromise` variable; subsequent calls return same promise
       - `importMapsLibrary(name: string): Promise<unknown>` — calls `(window as any).google.maps.importLibrary(name)`
       - Export a `_resetForTesting()` function that sets `loaderPromise = null` (for test isolation)

    2. Create `packages/widget/src/maps/autocomplete.ts`:
       - Module-level `sessionToken: any = null`
       - `fetchSuggestions(input: string): Promise<PlacePredictionResult[]>`
         - Calls `importMapsLibrary('places')` to get `AutocompleteSuggestion` and `AutocompleteSessionToken`
         - If `sessionToken` is null, create new `AutocompleteSessionToken()`
         - Call `AutocompleteSuggestion.fetchAutocompleteSuggestions({ input, sessionToken, language: 'en-US', region: 'us' })`
         - No `includedPrimaryTypes` filter per research Open Question 2 recommendation
         - Map results to `PlacePredictionResult[]`, storing original suggestion object in `raw`
         - Return empty array if input is empty or fewer than 3 chars
       - `resolvePlaceLocation(suggestion: PlacePredictionResult): Promise<SelectedPlace>`
         - Call `suggestion.raw.placePrediction.toPlace()` then `place.fetchFields({ fields: ['formattedAddress', 'location'] })`
         - Set `sessionToken = null` after fetchFields completes (ends billing session)
         - Return `{ formattedAddress, lat, lng }`
       - Export `_resetSessionForTesting()` that sets `sessionToken = null`
       - Use 05-RESEARCH.md Pattern 2 as reference but match the interface from types.ts

    3. Create `packages/widget/test/maps-loader.test.ts`:
       - Mock `document.head.appendChild` and `document.createElement`

       - Test: importMapsLibrary delegates to `google.maps.importLibrary`
       - Use `_resetForTesting()` in beforeEach for isolation

    4. Create `packages/widget/test/autocomplete.test.ts`:

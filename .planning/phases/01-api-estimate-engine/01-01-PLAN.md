    - Test: 1000 sqft, flat pitch, architectural shingles, default config -> estimateLow=4000, estimateHigh=5800 (rounding check)
    - Test: boundary rounding -- values ending in exactly X50 round up to next $100
    - Test: all three material types produce different ranges (3-tab cheapest, standing-seam-metal most expensive)
    - Test: steep pitch produces higher estimate than flat pitch for same inputs
    - Test: disclaimer string contains "estimate only"
    - Test: custom PricingConfig overrides are used instead of defaults when provided
  </behavior>
  <action>
    RED phase first: Create `packages/api/test/engine.test.ts` with all tests listed in behavior. Tests import calculateEstimate and default config values. Run tests -- they must FAIL (functions don't exist yet).

    GREEN phase: Create `packages/api/src/engine/defaults.ts`:
    - DEFAULT_MATERIALS: { '3-tab': { costLow: 3.50, costHigh: 4.75 }, 'architectural': { costLow: 4.00, costHigh: 5.75 }, 'standing-seam-metal': { costLow: 12.00, costHigh: 18.00 } }
    - DEFAULT_PITCH_MULTIPLIERS: { flat: 1.00, low: 1.05, medium: 1.12, steep: 1.25 }
    - DEFAULT_COMPLEXITY_MULTIPLIER: 1.0 (v1 -- no complexity selection, that is v2 EST-07)

    Create `packages/api/src/engine/calculate.ts`:
    - Import PricingConfig and EstimateResult from ../types
    - Export function `calculateEstimate(sqft: number, pitch: string, material: string, config: PricingConfig): EstimateResult`
    - Formula: rawLow = sqft * pitchMultipliers[pitch] * complexityMultiplier * materials[material].costLow
    - Formula: rawHigh = sqft * pitchMultipliers[pitch] * complexityMultiplier * materials[material].costHigh
    - Round: Math.round(raw / 100) * 100
    - Disclaimer: "This is an estimate only. Final pricing requires an on-site inspection."
    - Return { estimateLow, estimateHigh, disclaimer }

    Run tests -- they must all PASS.

    Note: complexity_multiplier is 1.0 for v1 per research recommendation. The formula includes it for future-proofing (v2 EST-07) but the API does not accept it as input.
  </action>

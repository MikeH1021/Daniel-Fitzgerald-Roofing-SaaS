  <action>
    1. **Schema change** (`packages/api/src/db/schema.ts`):
       - Add `role` column to `companies` table: `role: text('role').default('company-admin').notNull()` with allowed values 'super-admin' | 'company-admin'.
       - Add `slug` column to `companies` table if not present (it exists but nullable; leave as-is).

    2. **Session update** (`packages/api/src/auth/session.ts`):
       - Change `validateSession` to return `{ companyId: string; role: string } | null` instead of just `companyId: string | null`. Join on `companies` to fetch role.

    3. **Auth middleware** (`packages/api/src/middleware/auth.ts`):
       - Expand `AdminVars` to `{ companyId: string; role: 'super-admin' | 'company-admin' }`.
       - `authMiddleware` now sets both `companyId` and `role` from `validateSession` result.
       - Export a new `superAdminOnly` middleware that checks `c.get('role') === 'super-admin'` and returns 403 if not.
       - Export a new `companyAccessGuard` middleware that checks: if role is `super-admin`, allow. If role is `company-admin`, check that the `:companyId` route param matches `c.get('companyId')`, else return 403.

    4. **CSRF middleware** (`packages/api/src/middleware/csrf.ts` - new file):
       - For non-GET/HEAD/OPTIONS requests, check `Origin` header matches the request host OR check for `X-CSRF-Token` header matching a value derived from the session token (use first 16 chars of session token as CSRF token for simplicity since sessions are already random nanoid(32)).
       - Export `csrfMiddleware`. Return 403 with `{ error: 'CSRF validation failed' }` on failure.
       - Exempt `/api/admin/login` and `/api/admin/setup` (no session yet).
       - Also add a `GET /api/admin/csrf-token` endpoint (protected by auth) that returns `{ token: <first 16 chars of session cookie> }` so the frontend can fetch it.

    5. **Rate limiting** (`packages/api/src/middleware/rate-limit.ts` - new file):
       - Add `LOGIN_RATE_LIMITER` binding to `Bindings` type in `types.ts`.
       - Add rate limit binding in `wrangler.toml`: `[[unsafe.bindings]]` with name `LOGIN_RATE_LIMITER`, type `ratelimit`, limit 5, period 60.
       - Export `loginRateLimiter` middleware that uses the binding. Key by client IP (`c.req.header('cf-connecting-ip') || c.req.header('x-forwarded-for') || 'unknown'`).
       - If binding unavailable (local dev/test), fall back to an in-memory Map with IP -> { count, resetAt } for testability. Return 429 `{ error: 'Too many login attempts. Try again later.' }` when limit exceeded.

    6. **Wire into routes** (`packages/api/src/routes/admin.ts`):
       - Apply `loginRateLimiter` to `POST /admin/login`.
       - Apply `csrfMiddleware` after `authMiddleware` for all protected routes.
       - Apply `superAdminOnly` to `POST /admin/companies` (create company) and `GET /admin/companies` (list all companies).
       - Apply `companyAccessGuard` to all `/admin/companies/:companyId/*` routes.
       - Also add `GET /admin/csrf-token` protected route returning the token.
       - Add `GET /admin/me` protected route returning `{ companyId, role, name }` (replaces using `/admin/settings` for session checking on the frontend).

    7. **Update tests** (`packages/api/test/admin.test.ts`):
       - Update `seedAdminData` to include `role` column in CREATE TABLE.
       - Add test for RBAC: seed a super-admin company and a company-admin company. Verify super-admin can list companies, company-admin gets 403.
       - Add test for company-admin accessing own vs other company's settings.
       - Add test for CSRF: state-changing request without Origin returns 403, with matching Origin succeeds.
       - Add test for rate limiting: 6 sequential failed logins, 6th returns 429.

1. **Logo routes (packages/api/src/routes/admin.ts):** Add two routes to the existing admin router:
   - `POST /logo` (protected by authMiddleware): Parse formData, get 'logo' File. Validate type (image/png, image/jpeg, image/webp, image/svg+xml) and size (max 1MB = 1048576 bytes). Store in R2 at key `{companyId}/logo{ext}` where ext is derived from MIME type. Update company.logoUrl to `/api/logos/${companyId}`. Return {logoUrl}.
   - Mount a separate public logo route on the main app (in index.ts): `GET /api/logos/:companyId` -- reads `{companyId}/logo*` from R2 (list with prefix, take first), streams body back with Content-Type from R2 metadata. Returns 404 if not found.

2. **Logo tests (packages/api/test/admin.test.ts):** Add test for logo upload (create a minimal PNG blob in the test, send as FormData, verify 200 response with logoUrl). Add test for rejection of oversized file and wrong type.

3. **Admin UI package (packages/admin/):** Create a minimal Preact SPA:

   - `package.json`: name "roofing-admin", dependencies on preact, devDependencies on vite, @preact/preset-vite, typescript. Scripts: "dev" (vite), "build" (vite build).
   - `tsconfig.json`: Standard Preact config with jsx: "react-jsx", jsxImportSource: "preact".
   - `vite.config.ts`: Preact preset, output to dist/.
   - `index.html`: Minimal HTML shell with `<div id="app">` and script type="module" pointing to src/main.tsx.
   - `src/main.tsx`: Render `<App />` into #app.

4. **API client (packages/admin/src/api.ts):** Create a thin fetch wrapper that:
   - Uses relative URLs (same origin -- no CORS issues)
   - Always includes `credentials: 'include'` for cookies
   - Has methods: `login(email, password)`, `getSettings()`, `updateSettings(data)`, `uploadLogo(file)`, `getPricing()`, `updatePricing(overrides)`, `getEmbedCode()`, `logout()`

5. **Components:**
   - `LoginForm.tsx`: Email + password fields, submit calls api.login(). On success, sets logged-in state in parent.
   - `BrandingSettings.tsx`: Shows current logo (img tag from logoUrl), file input for new logo upload (calls api.uploadLogo), native `<input type="color">` for primaryColor (calls api.updateSettings on change). Shows save confirmation.
   - `PricingSettings.tsx`: Table of materials (3-tab, architectural, standing-seam-metal) with editable costLow/costHigh fields. Pitch multiplier section with editable fields for each pitch level. Save button calls api.updatePricing. Load current values on mount via api.getPricing.
   - `EmbedCode.tsx`: Fetches embed code from api.getEmbedCode. Displays in a `<code>` block. "Copy" button uses navigator.clipboard.writeText. Shows "Copied!" feedback for 2 seconds.

6. **App.tsx:** If not logged in, show LoginForm. If logged in, show a simple tabbed layout with four sections: Branding, Pricing, Embed Code, and a Logout button. Use simple state (useState or signals) for active tab. Minimal inline CSS -- clean and functional, not fancy.

7. Run `cd packages/api && npx vitest run` to verify logo route tests pass alongside all existing tests.

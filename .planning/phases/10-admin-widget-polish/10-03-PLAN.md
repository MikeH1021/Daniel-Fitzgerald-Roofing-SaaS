1. In client.ts, update the `submitEstimate` function error handling:
   - Instead of `throw new Error('Failed to submit estimate')`, parse the response body for specific errors.
   - Replace the error block with:
     ```typescript
     if (!res.ok) {
       let message = 'Something went wrong. Please try again.';
       try {
         const body = await res.json();
         if (body.error) {
           message = body.error;
         }
         if (body.details && Array.isArray(body.details)) {
           message = body.details.map((d: { field: string; message: string }) => d.message).join('. ');
         }
       } catch {
         // If response body is not JSON, use status-based message
         if (res.status === 429) {
           message = 'Too many requests. Please try again later.';
         } else if (res.status >= 500) {
           message = 'Server error. Please try again later.';
         }
       }
       throw new Error(message);
     }
     ```
   - This handles three cases:
     - 429 rate limit: shows "Too many requests. Please try again later." (from API body)
     - 400 validation: shows specific field messages from details array
     - 500+ server error: shows "Server error. Please try again later."
